# 前端开发指南

Guideflow 的 Web 前端：面向 NCCN B 细胞淋巴瘤（默认 `disease_scope: auto`）的指南问答界面。原生 HTML / CSS / JS，无构建步骤。

后端默认 **agentic** 路由：SSE 流除答案 token 外，还会推送 **`status` 进度事件**，前端以 Cursor 风格的步骤列表实时展示检索/看图/生成阶段。

架构与 API 协议详见 [`docs/技术实现.md`](../docs/技术实现.md)；环境搭建见 [`docs/环境配置.md`](../docs/环境配置.md)。

## 文件结构

| 文件 | 说明 |
|------|------|
| `index.html` | 主界面（两栏布局 + 鉴权弹窗） |
| `app.js` | 状态管理、API 调用、SSE 流式、步骤 UI、渲染逻辑 |
| `styles.css` | 样式（含 `.msg-row`、`.msg-actions`、`.status-steps`、`.ai-disclaimer`） |
| `share.html` | 公开分享只读页（`?token=`） |
| `vendor/marked.min.js` | Markdown 解析 |
| `vendor/purify.min.js` | HTML XSS 消毒 |
| `package.json` | 可选：`npm run dev` 等价于本地静态服务 |

## 本地启动

**后端（:8001）**

```bash
# 建议在项目 .venv 中运行
.\.venv\Scripts\python.exe -m uvicorn backend.api.server:app --reload --host 127.0.0.1 --port 8001
```

**前端（:5173）**

```bash
# 方式 A：Python 静态服务
python -m http.server 5173 -d frontend --bind 127.0.0.1

# 方式 B：在 frontend 目录用 npm 脚本
cd frontend && npm run dev
```

浏览器打开 http://127.0.0.1:5173/ 。

也可通过后端挂载访问 http://127.0.0.1:8001/app/ （`backend/api/server.py` 将 `frontend/` 挂到 `/app`）。

API 地址自动选择：经 `/app/` 或端口 `8001` 打开时用同源（空字符串）；本地 `5173` 静态页默认 `http://127.0.0.1:8001`。可在 `index.html` 或运行时通过 `window.API_BASE` 覆盖。分享页还支持 URL 参数 `?api=` 指定后端。

### 相关环境变量（后端）

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | 默认 `sqlite:///data/app.db` |
| `AUTH_SECRET` | 会话 HMAC 密钥（**生产必设**；未设时开发环境会写入 `data/.auth_secret`） |
| `AUTH_ALLOW_PASSWORD_RESET` | 默认 `true`：允许邮箱直重置密码（本地演示）；生产请设 `0` |
| `CORS_ORIGINS` | 逗号分隔白名单，默认含 `127.0.0.1:5173` |
| `COOKIE_SECURE` | 生产 HTTPS 设 `true` |
| `QWEN_API_KEY` / `VLM_API_KEY` | 模型密钥（仅服务端，勿注入前端） |
| `ROUTING_MODE` | 默认 `agentic`；前端无需感知，但影响 `status` 事件内容 |

## SSE 流式协议

`POST /api/ask` 在 `stream: true`（默认）时返回 `Content-Type: text/event-stream`。每行事件格式：

```text
data: {"type":"...", ...}

```

（空行分隔多条事件。）

### 事件顺序

```text
meta → status* → token* → final
```

| `type` | 前端处理 | 说明 |
|--------|----------|------|
| `meta` | 当前未单独处理 | 含 `routing_mode`、`disease_scope`、`run_id` 等 |
| `status` | **`consumeSSE` 追加到 `assistant.statusSteps` 并重绘** | 智能体/生成进度 |
| `token` | 增量写入答案 Markdown | 首个 token 后 `pending=false`，开始打字流 |
| `final` | `finalizeAssistant()` | 完整 payload，含 sources、figures、trace |
| `error` | 未专门分支 | 服务端异常时可能出现 |

### `status` 事件字段

```json
{
  "type": "status",
  "stage": "search",
  "label": "检索指南中：「DLBCL 一线治疗」",
  "detail": { "tool": "search_guidelines", "kind": "flowchart", "query": "..." }
}
```

常见 `stage` 值（与后端 `agent_tools.status_event` 一致）：

| `stage` | 含义 |
|---------|------|
| `planning` | 智能体开始规划 |
| `search` | BM25 检索指南（`search_guidelines` 或补充检索） |
| `graph` | 查询知识图谱（`query_graph`） |
| `view_pages` | 按需读取流程图页（`view_pages`） |
| `direct` | 闲聊 / 通用医学直接回复（`respond_directly`） |
| `fallback` | 回退线性检索 |
| `generate` | 进入 LLM/VLM 生成回答 |

## Cursor 风格步骤 UI

生成进行中（`message.pending === true`）时，`renderMessage()` 渲染：

1. **`.typing-indicator`**：转圈 + **当前步骤文案**（`statusSteps` 最后一项的 `label`，无步骤时显示「正在检索与生成回答…」）
2. **`.status-steps`**：纵向步骤列表，类似 Cursor Agent 的执行轨迹

```html
<ul class="status-steps">
  <li class="done"><span class="dot"></span><span>规划中…</span></li>
  <li class="done"><span class="dot"></span><span>检索指南中：「…」</span></li>
  <li class="active"><span class="dot"></span><span>读取流程图：BCEL-3</span></li>
</ul>
```

样式规则（`styles.css`）：

- `li.done`：已完成步骤，半透明
- `li.active`：当前步骤，加粗 + 品牌色圆点光晕
- `.dot`：7px 圆点指示器

### `consumeSSE` 实现要点

路径：`app.js` → `consumeSSE(resp, assistant)`

1. 创建 pending 助手消息时初始化 `statusSteps: []`（见 `askQuestion`）。
2. 解析 SSE `data:` JSON；`event.type === 'status'` 时：
   - `assistant.statusSteps.push({ stage, label, detail })`
   - 调用 `paintStatus()`：仅重绘**最后一条** pending 助手气泡（避免全页闪烁）。
3. `event.type === 'token'`：`pending = false`，增量渲染 Markdown。
4. `event.type === 'final'`：调用 `finalizeAssistant()`，写入完整 `payload` 并 `renderChat()`。

非 SSE 响应（`Content-Type` 非 `text/event-stream`）时降级为一次性 `resp.json()`。

## 功能清单

### 布局与导航

- **两栏**：左侧栏 + 主对话区（已移除原右侧证据栏）
- 侧栏可收起 / 展开；状态写入 `localStorage`（`gf_sidebar_collapsed`）
- 「开启新对话」位于侧栏顶部
- 顶栏品牌区 + 隐蔽「⚙ 工具」菜单

### 左侧栏 · 历史

- 登录后从 `GET /api/conversations` 拉取列表
- 未登录时从 `sessionStorage`（`gf_local_conversations`）读取本页本地会话列表
- 按 **今天 / 7 天内 / 30 天内 / 更早** 分组展示
- 点击条目打开会话；当前会话高亮
- 条目「⋯」：**重命名**、**删除**（本地改 sessionStorage；已登录走 `PATCH` / `DELETE`）
- 无会话时提示「暂无历史对话」
- 会话过期（401）时提示重新登录

### 空状态与快捷提问

- 无消息时展示引导文案 + **4 条示例问题**（可点击直接提问）
- 示例涵盖一线治疗、亚型、TP53、CNS prophylaxis 等

### 对话与输入

- **真流式**：`POST /api/ask` 带 `stream: true`，消费 SSE 事件 **`status` / `token` / `final`**
- 非 SSE 响应时自动降级为一次性 JSON
- 请求携带 `conversation_id`（已登录且续聊时；本地 `local-*` id 不传给后端）、`trace: true`、近 **6 轮** `history`
- 生成中显示步骤 UI + 加载转圈；进行中可因新提问 `AbortController` 取消上一轮
- 输入框：**回车发送**、**Shift+回车换行**、高度随内容自适应（最高约 160px）
- 输入框可 **展开 / 收起**（展开固定约 180px）
- **发送时立即清空输入**并收起展开状态（不等待 SSE 结束）

### 答案渲染

- Markdown（marked）+ DOMPurify 消毒
- 优先按 `answer_paragraphs` 分段；否则用 `answer_markdown`
- 流程图按 `anchor_paragraph` **内联**到对应段落；无锚点的图归入文末「相关流程图」
- 支持 **compact / full** 双视图；点击「放大（含脚注）」打开 lightbox
- 行内 `[Sn]` / `[Gn]` / 参考文献 `[n]` 渲染为可交互引用按钮
- 引用 **悬浮卡**：展示来源摘要；`See All` 滚动到文末 References；点击引用也可跳转
- 正文末、**References 之前**展示灰色斜体提示：`本回答由 AI 生成，内容仅供参考，请仔细甄别`（`.ai-disclaimer`）
- 文末可折叠 **References**（合并 `sources` + `attached_references`，含 PMID / DOI 链接）
- 助手 tag 随 `answer_kind` 切换（证据约束 / 非指南内容 / 闲聊）

### 消息布局与操作条（气泡外）

DOM 结构：`.msg-row`（整行）→ 内层 `.message`（气泡正文）→ 气泡外 `.msg-actions`（操作栏）。`data-msg-idx` 挂在 `.msg-row` 上。

| 侧 | 布局 | 操作 |
|----|------|------|
| 用户 | `.msg-row.user` 右对齐；操作条贴气泡**右下外侧** | 复制问题、编辑问题；多版本时 `‹ n / m ›` |
| 助手 | `.msg-row.assistant` 操作条**左对齐**（不顶到行尾） | 复制、重说、赞/踩、分享；多版本时 `‹ n / m ›` |

#### 编辑问题

1. 点击铅笔进入编辑态（气泡内文本框 + 取消 / 发送）。
2. 发送改写后的问题：截断该轮之后的消息；当前气泡只显示**新问题**（旧文被覆盖）。
3. 前端在本会话内保留版本：`user.variants[]` + 配对助手的 `variantAnswers[][]`。
4. 用户侧与助手侧均可点 `‹ n / m ›` 切回旧问 + 对应旧答（`syncTurnView`）。

#### 重说

- 不再丢弃旧回答：把当前回答快照写入 `variantAnswers[variantIndex]`，再生成新回答。
- 同一问下多条回答时，助手侧显示回答版本 `‹ n / m ›`（`prev-answer` / `next-answer`）。
- 若该轮已因编辑产生多个问题版本（`variants.length > 1`），助手侧优先显示**问题版本**切换（与用户侧同步）。

#### 操作表

| 操作 | 行为 | 前置条件 |
|------|------|----------|
| 复制（用户） | 复制问题原文 | 无 |
| 编辑（用户） | 气泡内改写后重问，保留可切换历史版本（仅前端会话） | 非生成中 |
| 复制（助手） | 复制 `answer_markdown` 纯文本，图标短暂变为 ✓ | 无 |
| 重说 | 同问再生成，保留旧回答版本可切换 | 非生成中 |
| 点赞 / 点踩 | 本地更新；已登录且有 `id` 时 `POST /api/messages/{id}/feedback`，可取消 | 反馈同步需登录落库 |
| 分享 | 创建分享 token，复制 `share.html?token=...&api=...` 链接 | 需登录且当前对话已保存到云端 |

版本字段会写入 `snapshotMessagesForStore`（草稿 / 本地会话）；从服务端重新加载的旧会话无版本字段时按单版本兼容。后端仍为线性追加，无分支 API。

### 调试工具（⚙）

对**最近一次完整回答**的 payload 打开抽屉：

- **Trace** — JSON 格式化 trace（含 `agent_steps`、`retrieval_stages`）
- **证据来源** — `sources` 卡片列表
- **图谱路径** — `graph_triples` 三元组
- **流程图** — `figures` 缩略图（可放大）

### 鉴权

- 邮箱 **注册 / 登录 / 退出**（Cookie 会话，`credentials: 'include'`）
- 弹窗支持 **忘记密码** → `POST /api/auth/reset-password`（本地直重置，无邮件）
- 密码框 **显示 / 隐藏** 切换
- 点击遮罩关闭弹窗（仅在遮罩上按下并松开，避免拖选文字误关）
- 未登录可问答并维护本页多条本地历史；登录后侧栏改用服务端历史，后续提问写入云端（不 bulk 导入本地旧会话）

### 本地持久化（未登录 / 刷新）

| 存储 | 键 | 内容 |
|------|-----|------|
| `localStorage` | `gf_sidebar_collapsed` | 侧栏折叠 |
| `localStorage` | `gf_active_conversation_id` | 上次打开的**服务端**会话 ID（登录后） |
| `sessionStorage` | `gf_chat_draft` | 当前页草稿：消息列表（可含 `variants` / `variantAnswers`）、`lastPayload`、临时 `conversationId` |
| `sessionStorage` | `gf_local_conversations` | 未登录多条本地会话（`local-*` id、title、messages） |

刷新后：已登录则优先恢复 `gf_active_conversation_id` 对应会话；否则尝试恢复 `sessionStorage` 草稿，并展示本地会话列表。本地历史仅当前标签页有效（关闭标签即清）。

### 分享页（`share.html`）

- 只读展示：`GET /api/shared/{token}`
- URL：`share.html?token=...`，可选 `&api=` 覆盖后端地址
- 渲染用户消息 + AI Markdown 答案；文末含 AI 生成提示；无消息操作条、无步骤 UI
- 失效或缺少 token 时显示友好错误

## API 依赖（前端调用）

| 方法 | 路径 | 用途 |
|------|------|------|
| `POST` | `/api/ask` | 问答（SSE：`status` + `token` + `final`） |
| `GET` | `/api/auth/me` | 当前用户 |
| `POST` | `/api/auth/login` | 登录 |
| `POST` | `/api/auth/register` | 注册 |
| `POST` | `/api/auth/logout` | 退出 |
| `POST` | `/api/auth/reset-password` | 重置密码 |
| `GET` | `/api/conversations` | 历史列表 |
| `GET` | `/api/conversations/{id}` | 打开会话 |
| `PATCH` | `/api/conversations/{id}` | 重命名 |
| `DELETE` | `/api/conversations/{id}` | 删除 |
| `POST` | `/api/conversations/{id}/share` | 创建分享 |
| `POST` | `/api/messages/{id}/feedback` | 点赞 / 点踩 |
| `GET` | `/api/shared/{token}` | 分享只读 |
| `GET` | `/api/images/{filename}` | 页图 / 裁剪图（由 payload URL 引用） |

### `AskRequest` 请求体

```json
{
  "question": "DLBCL 一线治疗？",
  "stream": true,
  "trace": true,
  "conversation_id": null,
  "history": [
    { "role": "user", "content": "..." },
    { "role": "assistant", "content": "..." }
  ]
}
```

无 `embedding` 字段；检索模式由服务端 `routing_mode` 决定。

## 安全注意（上云）

1. 轮换 `.env` 中的模型 API Key（勿提交 git）
2. 设置强随机 `AUTH_SECRET`；生产关闭密码直重置（`AUTH_ALLOW_PASSWORD_RESET=0`），`COOKIE_SECURE=true`，收紧 `CORS_ORIGINS`
3. 建议反代同源托管静态页 + `/api`；SQLite 单 worker 或改 Postgres
4. 前端已对 Markdown 消毒；勿把密钥注入 `window.API_BASE` 以外的配置
