# DLBCL 专病问答 Agent 项目需求分析及初步思路

## 1. 需求总结

目标是构建一个面向医生群体的弥漫大 B 细胞淋巴瘤（DLBCL）专病问答 Agent。它不是泛泛回答医学常识，而是围绕权威指南和证据来源，给出更专业、可追溯、结构化的临床辅助回答。

核心使用场景：

- 医生主动询问 DLBCL 诊断、分期、检查、预后、治疗路径、随访或指南依据等问题。
- AI 根据 NCCN 指南中的决策流程、discussion 文本和 reference 信息生成回答。
- 回答需要明确说明依据来自指南的哪个部分，适用条件是什么，哪些内容指南有明确支持，哪些内容属于不确定或超出指南范围。

当前主要数据源：

- 《NCCN Clinical Practice Guidelines in Oncology: B-Cell Lymphomas, Version 3.2026》。
- 前半部分（第62页开始是DLBCL相关内容）：类似树状结构的诊疗流程图，包含大量蓝色跳转链接。例如 workup 页面跳转到 first-line therapy。
- 后半部分（第255页到第296页）：discussion 文本（255-280页）和对应的references（281-296页），包含疾病定义、诊断、分型、预后、检测、证据解释等等及文献引用。

期望效果：

- 比直接上传 PDF 给通用大模型更稳定地找到答案。
- 回答必须带引用来源，避免“看似正确但无法溯源”。
- 对诊疗流程类问题，能够沿 NCCN 的结构路径回答，而不是只做语义相似段落拼接。
- 对 discussion类问题，能够提取文字证据并给出对应的参考文献。

## 2. 现有情况与问题

如果直接把文件“（2026.V3）NCCN 临床实践指南：B 细胞淋巴瘤”上传给大模型，例如 DeepSeek，输出效果不好：

- 找不到结果，尤其是具体突变位点、跳转路径、流程图节点相关问题。
- 没有可靠引用来源，医生难以判断答案是否可信。
- PDF 前半部分是带链接的流程图，直接解析为普通文本后会丢失“节点、条件、跳转关系”。
- 指南后半部分 discussion 与 reference 虽然是文本，但需要和前半部分流程图建立对应关系。
- 通用大模型容易把指南内证据、指南外知识和模型常识混在一起，导致医学安全风险。

### 示例

User:

> TP53 R248Q 在 DLBCL 的预后意义是什么？

DeepSeek:

> 根据您提供的 2026.V3 版 NCCN 指南，该指南没有直接提及或讨论 TP53 R248Q 这个特定突变位点的预后意义。
> 不过，指南中确实有关于 TP53 突变在弥漫性大 B 细胞淋巴瘤（DLBCL）中预后意义的相关描述，可以作为参考：
>
> 预后相关性：指南指出，TP53 突变与 MYC 重排同时存在时，与更差的预后相关。¹
>
> 分子分型关联：TP53 突变在具有“暗区”基因表达特征的大 B 细胞淋巴瘤中更为常见，这类淋巴瘤通常伴有 MYC 和 BCL2（或 BCL6）重排（即双打击淋巴瘤）。²
>
> 对您的建议：
>
> 综合评估：TP53 R248Q 是一个已知的、常影响 p53 蛋白功能的错义突变。您需要结合其他临床因素（如国际预后指数 [IPI] 评分、细胞起源 [COO] 分型、MYC/BCL2 重排状态等）来综合评估其对特定患者预后的影响。
>
> 咨询专家：鉴于指南未提供针对该特定突变位点的明确信息，建议您咨询血液科或肿瘤科专家，他们可以结合最新的临床研究数据和患者的具体情况，提供更具个体化的解读和建议。
>
> 如果您想进一步了解指南中关于高危 DLBCL 的治疗推荐，我可以为您提供相关信息。
>
> 本回答由 AI 生成，内容仅供参考，请仔细甄别。

这个回答的问题在于：虽然承认指南没有直接提及 TP53 R248Q，但没有给出可核查的指南页码、章节、原文位置和 reference；同时也没有明确区分“指南内信息”和“模型外部医学知识”。

## 3. 当前设想技术路线

当前考虑的技术路线是：

> 按 NCCN DLBCL 决策树做语义 + 结构感知切片，使用 Hierarchical-Filtered-RAG，最后加一个校验和对抗 Agent 检查输出结果。

结论：这个方向总体可行，而且比“直接把 PDF 丢给大模型”更适合当前任务。但它需要进一步明确“结构化知识建模”和“医学证据校验”两个关键环节，否则容易变成普通 RAG 的复杂版本，仍然无法稳定处理 NCCN 前半部分的流程图和跳转路径。
