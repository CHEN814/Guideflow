# Guideflow 华为云部署指南（新手版）

面向**第一次使用华为云**、需要把 Guideflow 部署成「可点击链接试用」的同学。按本文做完后，医生可用浏览器打开类似：

```text
http://你的公网IP:8001/app/
```

进行注册/登录与问答试用。

> **部署在做什么？**  
> 你本机上的网站只在 `127.0.0.1` 能打开；部署就是把同一套后端 + 前端放到一台 24 小时在线的云服务器上，并暴露公网地址。

相关文档：[环境配置](./环境配置.md) · [运维手册](./运维手册.md)（部署完成后的日常命令、查库、更新代码） · [前端开发指南](../frontend/开发指南.md)

---

## 0. 你将得到什么 / 需要买什么

| 目标 | 做法 |
|------|------|
| 给医生一个网页链接 | 买一台 **弹性云服务器 ECS** + **公网 IP** |
| 小规模试用（几人～十几人） | **一台 ECS 足够**，不必买 GPU、K8s、RDS、对象存储 |
| 大模型问答 | 继续用已有的 `QWEN_API_KEY` / `VLM_API_KEY`（费用在通义等侧，不是华为云 GPU） |

**本文推荐的最简方案（方案 A）：**

1. 云服务器上只跑一个进程：`uvicorn`（FastAPI）
2. 前端由后端自带挂载提供：访问 `/app/`
3. 安全组放行 **8001**（以及 SSH 的 **22**）
4. 暂不强制域名 / HTTPS（试用阶段可用 IP；正式对外建议再加 nginx + HTTPS）

文末另有 **方案 B（nginx 反代 80 端口）**，链接更短、更像正式站点。

## 1. 开始前请准备

在打开华为云控制台之前，确认手头有：

| 材料 | 说明 |
|------|------|
| 华为云账号 | 导师已提供且已充值 |
| 本机可 SSH / 传文件 | Windows 可用 PowerShell，或安装 [WinSCP](https://winscp.net/) |
| 项目代码 | GitHub：`https://github.com/CHEN814/Guideflow.git`（若仓库为私有，需准备访问方式；也可本机打包上传） |
| NCCN / CSCO PDF 或已构建好的 `data/` | 服务器上要能构建或直接拷贝知识库（含可选 `csco_bm25_index.pkl`） |
| API Key | `QWEN_API_KEY`；流程图能力另需 `VLM_API_KEY` |
| 约 30～90 分钟 | 含购机、装环境、建索引、联调 |

**规格建议（试用）：**

| 项 | 建议 |
|----|------|
| 镜像 | **Ubuntu 22.04** 64 位 |
| CPU / 内存 | **2 核 4GB** 可起步；更稳妥 **2～4 核 8GB**（见环境配置文档） |
| 系统盘 | **20GB** 可起步（默认轻量依赖）；若取消注释安装 torch 等重型包，建议 **40GB** 及以上 |
| 带宽 | **1～5 Mbps** 够小规模试用 |
| GPU | **不需要**（默认 BM25，模型走云端 API） |

## 2. 登录华为云控制台

1. 打开华为云官网：<https://www.huaweicloud.com/>
2. 右上角 **登录**，使用导师提供的账号
3. 登录后进入 **控制台**（Console）
4. 页面右上角确认：
   - **区域**（Region）：选离你近、且有库存的，例如「华东-上海」「华北-北京」  
   - 之后创建的 ECS、安全组都要在**同一区域**，否则互相看不见

> 账号若提示实名认证、完善信息，按页面完成后再买资源。

## 3. 购买弹性云服务器（ECS）

### 3.1 找到购买入口

控制台搜索框输入：**弹性云服务器** 或 **ECS** → 进入服务页 → **购买弹性云服务器**。

不同控制台皮肤菜单名称可能略有差异，以搜索「ECS」为准。

### 3.2 关键配置（按界面逐步选）

下面按常见向导说明；若某一步名称不同，对照含义选择即可。

1. **计费模式**
   - 短期试用：可选 **按需计费**（用完可关机/释放，注意费用规则）
   - 连续试用几周：可选 **包年包月**（通常更便宜）
2. **区域**：与第 2 节所选区域一致
3. **规格**：按上文「2 核 4G / 8G」筛选；系列选通用计算即可
4. **镜像**：公共镜像 → **Ubuntu** → **22.04**
5. **存储**：系统盘 40GB+（高 IO 非必须）
6. **网络**
   - 使用默认 VPC / 子网即可（新手不必新建复杂网络）
   - **弹性公网 IP**：选择 **现在购买** / **自动分配**（必须有公网 IP，否则外网打不开）
   - 带宽按建议选择
7. **安全组**（很重要）
   - 可先选默认安全组，下一节再改规则；或新建安全组并预先加规则（见第 4 节）
8. **登录凭证**
   - 推荐：**密码**（新手好记）
   - 设置 root 或默认用户（Ubuntu 一般为 `root` 或 `ubuntu`，以购买页说明为准）密码，**务必保存到本地密码管理器**
   - 若选密钥对：下载 `.pem` 私钥并妥善保管
9. 确认费用 → **立即购买** → 等待实例状态变为 **运行中**

### 3.3 记下两样东西

在 ECS 列表中找到刚买的机器，记下：

1. **弹性公网 IP**（形如 `123.60.x.x`）——这就是别人访问你网站的地址基础  
2. **登录用户名 + 密码**（或密钥文件路径）

下文用 `YOUR_EIP` 表示你的公网 IP。

## 4. 配置安全组（放行端口）

安全组 = 云上的防火墙。不放行则浏览器/SSH 都会超时。

1. 控制台打开该 ECS → **安全组** → 进入关联的安全组 → **入方向规则** → **添加规则**
2. 至少添加：

| 优先级 | 协议 | 端口 | 源地址 | 用途 |
|--------|------|------|--------|------|
| 1 | TCP | 22 | 建议先填你自己的公网 IP/32；试用也可临时 `0.0.0.0/0` | SSH 登录 |
| 1 | TCP | 8001 | `0.0.0.0/0` | Guideflow Web（方案 A） |
| （方案 B 再加） | TCP | 80 | `0.0.0.0/0` | HTTP |
| （可选） | TCP | 443 | `0.0.0.0/0` | HTTPS |

3. 保存后稍等几秒生效。

> 试用阶段常用 `0.0.0.0/0`（全世界可访问）。正式环境应收紧 SSH 来源 IP，并尽快上 HTTPS。

## 5. 从 Windows 登录服务器（SSH）

### 5.1 用 PowerShell（Windows 10/11 自带）

```powershell
ssh root@YOUR_EIP
```

若镜像默认用户是 `ubuntu`：

```powershell
ssh ubuntu@YOUR_EIP
```

- 第一次会问 `Are you sure you want to continue connecting` → 输入 `yes`
- 再输入密码（输入时不显示字符，属正常）

登录成功后提示符类似：`root@ecs-xxxx:~#`

### 5.2 若提示无法连接

按顺序检查：

1. ECS 是否为 **运行中**
2. 是否有 **弹性公网 IP**
3. 安全组是否放行 **22**
4. 本机网络是否屏蔽 22（公司网有时会拦；可换手机热点试）

### 5.3 （可选）用密钥登录

```powershell
ssh -i "C:\path\to\your-key.pem" ubuntu@YOUR_EIP
```

Linux 密钥权限需限制为仅本人可读；Windows 上若报权限错误，可在密钥文件属性里去掉其他用户的读取权限。

## 6. 服务器基础环境

以下命令均在 **SSH 登录后的 Linux 终端**执行。

### 6.1 更新系统并安装基础软件

```bash
sudo apt update
sudo apt install -y git curl wget build-essential
```

### 6.2 安装 Python 3.11（推荐）

Ubuntu 22.04 通常自带 3.10，也可用；若要 3.11：

```bash
sudo apt install -y python3 python3-venv python3-pip python3-dev
python3 --version
```

确认版本在 **3.10～3.12**。

### 6.3 创建项目目录

```bash
sudo mkdir -p /opt/guideflow
sudo chown -R "$USER":"$USER" /opt/guideflow
cd /opt/guideflow
```

## 7. 把代码放到服务器

任选一种方式。

### 方式 A：Git 克隆（仓库公开或已配置访问）

```bash
cd /opt/guideflow
git clone https://github.com/CHEN814/Guideflow.git .
# 若默认分支不是你开发用的分支，再切换，例如：
# git checkout guideflow-initial
```

> **注意末尾的 `.`**：表示克隆到当前目录 `/opt/guideflow`，而不是再套一层 `Guideflow/` 子目录。  
> 若写成 `git clone ... Guideflow.git`（没有 `.`），代码会落在 `/opt/guideflow/Guideflow/`，后续路径都要改。

> 仓库若为私有：在 GitHub 建 Fine-grained / Personal Access Token，用 HTTPS + Token 克隆；或改用本机上传。

**国内 ECS 常见失败：** `GnuTLS recv error (-110)` / `Connection timed out` — 华北等区域访问 GitHub 经常不稳定，**优先改用下面的方式 B（本机上传）**，成功率最高。

若仍想试 Git，可依次尝试：

```bash
# 1) 用镜像站（镜像可用时再试）
git clone https://mirror.ghproxy.com/https://github.com/CHEN814/Guideflow.git .

# 2) 或加大缓冲后重试官方地址
git config --global http.postBuffer 524288000
git clone https://github.com/CHEN814/Guideflow.git .
```

克隆失败时清理半成品再换方式：

```bash
cd /opt/guideflow
rm -rf Guideflow .git   # 仅当目录里只有失败的克隆残留时
```

### 方式 B：本机打包上传（不依赖 GitHub）

在**你的 Windows 项目根目录**（含 `backend/`、`frontend/`、`requirements.txt` 的目录）打包，注意不要打进本地 `.venv`，也不要把压缩包打进自己：

```powershell
# 在项目根目录执行（PowerShell）
# 若已有旧的 guideflow-src.tgz，先删掉，否则会报：Can't add archive to itself
Remove-Item -Force .\guideflow-src.tgz -ErrorAction SilentlyContinue

tar -czf guideflow-src.tgz --exclude=.venv --exclude=__pycache__ --exclude=.git --exclude=ecs-deploy --exclude=docs/临时文档 --exclude=guideflow-src.tgz .
```

- 压缩时终端可能**几秒到一两分钟没有新输出**，属正常；成功后会回到提示符，并生成 `guideflow-src.tgz`
- 若立刻出现 `Can't add archive to itself`，说明旧压缩包还在目录里，按上面先删除再打包

上传（把路径换成你的）：

```powershell
scp .\guideflow-src.tgz root@YOUR_EIP:/opt/guideflow/
```

服务器上解压：

```bash
cd /opt/guideflow
tar -xzf guideflow-src.tgz
```

也可用 WinSCP 图形界面拖拽整个项目文件夹到 `/opt/guideflow`（同样建议排除 `.venv`）。

### 确认目录结构

```bash
cd /opt/guideflow
ls
# 应能看到：backend  frontend  scripts  requirements.txt  config.yaml  docs 等
```

## 8. 安装 Python 依赖

```bash
cd /opt/guideflow
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

- 默认 `requirements.txt` 已注释掉 torch 等重型依赖，安装通常几分钟即可；缺失时服务端自动回退 hash 向量 + 词法重排
- 若取消注释启用 torch / sentence-transformers / chromadb 等，首次安装可能需 **10～30 分钟**，并占用更多磁盘
- 磁盘不足时 `pip` 会失败：扩容系统盘或清理空间后重试
- 验证：

```bash
python -c "import fastapi, uvicorn; print('ok')"
```

## 9. 配置密钥与生产环境变量

```bash
cd /opt/guideflow
nano .env
```

写入（把密钥换成真实值；`AUTH_SECRET` 用随机长串）：

```bash
QWEN_API_KEY=你的DashScope密钥
VLM_API_KEY=你的VLM密钥

# 生产必须设置，用于登录会话签名
AUTH_SECRET=请换成一长串随机字符例如openssl_rand_hex_32

# 关闭「仅邮箱即可重置密码」的演示能力（重要）
AUTH_ALLOW_PASSWORD_RESET=0

# Cookie：纯 HTTP 用 IP 访问时先保持 false；上了 HTTPS 再改为 true
COOKIE_SECURE=0
COOKIE_SAMESITE=lax

# 方案 A：浏览器会从公网 IP:8001 访问 API，需写入 CORS
# 把下面的 YOUR_EIP 换成真实公网 IP（两处都要改）
CORS_ORIGINS=http://YOUR_EIP:8001

# —— 可选：Neo4j 交互式图谱（Tools → Neo4j 图谱）——
# 未配置时主问答仍可用（JSON 图谱 + BM25）；配置后启用 /api/graph/*
# 导入：python scripts/import_kg_to_neo4j.py（需 APOC；驱动 neo4j 5.x）
# NEO4J_URI=bolt://127.0.0.1:7687
# NEO4J_USER=neo4j
# NEO4J_PASSWORD=change-me-strong-password
# NEO4J_DATABASE=neo4j
```

生成随机 `AUTH_SECRET` 可用：

```bash
openssl rand -hex 32
```

保存退出：`nano` 中 `Ctrl+O` 回车保存，`Ctrl+X` 退出。

> **不要**把 `.env` 提交到 Git，也不要发到微信群。

## 10. 前端 API 地址（一般不用手改）

页面会自动判断：

- 通过 **`/app/`** 打开（云上方案 A，或本机 `http://127.0.0.1:8001/app/`）→ 自动连**当前网站自己的接口**，不写死公网 IP  
- 本地用 **5173** 单独开页面 → 仍默认连 `http://127.0.0.1:8001`

因此 **`ecs-deploy` 同步后不必再改 `frontend/index.html`**。医生打开：

```text
http://YOUR_EIP:8001/app/
```

即可。换公网 IP 或以后改端口（只要仍用 `/app/`）通常也不用改前端。

仅在特殊场景需要覆盖时，可在页面加载前设置 `window.API_BASE`（例如显式写成 `http://YOUR_EIP:8001`）。方案 B（nginx 同域）下保持自动检测即可（路径含 `/app` 时同样走同源）。

## 11. 准备知识库与 BM25 索引

问答依赖 `data/` 下的知识库与索引。两种做法二选一。

### 做法 1：在服务器上构建（步骤完整，耗时更长）

1. 把 NCCN PDF、以及（可选）CSCO PDF 放到项目中（路径与 `config.yaml` / `.env` 一致，详见 [环境配置](./环境配置.md)）
2. 激活 venv 后执行：

```bash
cd /opt/guideflow
source .venv/bin/activate
python scripts/build_knowledge_base.py
python scripts/build_bm25_index.py
# CSCO 双源：
# python scripts/build_csco_knowledge_base.py
# python scripts/build_bm25_index.py --source csco
# Web「图谱路径」需要时再构建（仅 NCCN）：
# python scripts/build_knowledge_graph.py
# 或一站式（默认跳过 embedding；有 NEO4J_PASSWORD 时会导入 Neo4j）：
# python -m backend.scripts.build_knowledge_pipeline
# 仅导入已有 knowledge_graph.json：
# python scripts/import_kg_to_neo4j.py
```

### （可选）部署 Neo4j 以开启交互式图谱

主问答**不依赖** Neo4j。只有 Tools 里「Neo4j 图谱」邻域可视化需要。仓库根目录提供了带 **APOC** 的 compose 文件（云上推荐此方式；本机无 Docker 时见 [环境配置](./环境配置.md) 的 Windows 独立安装说明）：

```bash
# 安全组：7687（Bolt）仅建议内网/本机；7474（Browser）按需
cd /opt/guideflow
# 先编辑 docker-compose.neo4j.yml 里 NEO4J_AUTH 密码，再：
docker compose -f docker-compose.neo4j.yml up -d
# 在 .env 写入相同密码的 NEO4J_* 后：
python scripts/import_kg_to_neo4j.py
# 或：python -m backend.scripts.build_knowledge_pipeline   # 默认不建 FAISS
sudo systemctl restart guideflow
```

> 导入请用 `import_kg_to_neo4j.py` / `build_knowledge_pipeline`（`Neo4jGraphService` schema），勿与 `ask.py --import-neo4j` 混用。  
> **不要**默认安装 `sentence-transformers` / `faiss-cpu`。向量检索代码已保留但默认关闭；云端保持 BM25-only。

### 做法 2：本机已构建好，直接上传 `data/`（更快，推荐）

若你本机已经能正常问答，可把服务器需要的数据目录拷上去，例如：

- `data/processed/`
- `data/indexes/`（至少含 `bm25_index.pkl`；启用 CSCO 时还需 `csco_bm25_index.pkl`）
- 若流程图要用本地渲图缓存：按你环境中的 `page_image` / 缓存目录一并上传

本机示例：

```powershell
scp -r .\data root@YOUR_EIP:/opt/guideflow/
```

上传后在服务器确认：

```bash
ls -lh /opt/guideflow/data/indexes/bm25_index.pkl
ls -lh /opt/guideflow/data/processed/
```

## 12. 前台试跑（验证能通再做成开机自启）

```bash
cd /opt/guideflow
source .venv/bin/activate
python -m uvicorn backend.api.server:app --host 0.0.0.0 --port 8001
```

注意：

- 必须是 `--host 0.0.0.0`（绑定所有网卡），不能只用 `127.0.0.1`，否则外网进不来
- 终端保持运行；本机浏览器打开：

```text
http://YOUR_EIP:8001/health
```

应返回类似：`{"status":"ok"}`

再打开：

```text
http://119.3.182.137:8001/app/
```

应看到 Guideflow 页面。尝试注册/登录并提问。

若页面打开但登录/提问失败：多半是服务未起来、**第 9 节 `CORS_ORIGINS` 未含该地址**，或浏览器仍缓存了旧前端（强刷试试）。当前前端在 `/app/` 下会自动同源连 API，一般不必再改 `API_BASE`。

验证通过后，在 SSH 里 `Ctrl+C` 停掉前台进程，继续第 13 节做成后台服务。

## 13. 用 systemd 保持后台运行（关机重启也能起来）

### 13.1 创建服务文件

```bash
sudo nano /etc/systemd/system/guideflow.service
```

写入（用户名若不是 `root`，改 `User=` 与路径权限）：

```ini
[Unit]
Description=Guideflow QA API
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/guideflow
Environment=PATH=/opt/guideflow/.venv/bin
EnvironmentFile=/opt/guideflow/.env
ExecStart=/opt/guideflow/.venv/bin/python -m uvicorn backend.api.server:app --host 0.0.0.0 --port 8001
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### 13.2 启动并设置开机自启

```bash
sudo systemctl daemon-reload
sudo systemctl enable guideflow
sudo systemctl start guideflow
sudo systemctl status guideflow
```

状态应为 `active (running)`。

部署完成后的日常启停、日志、**查看 `data/app.db` 用户/对话记录**、代码更新等，见 **[运维手册](./运维手册.md)**。

涉及 `app.db` 结构变更的版本（例如消息树字段）上线前，请先按运维手册 **§2.4** 备份数据库，再同步代码并重启；`init_db()` 会自动迁移/回填，**不会**被 `ecs-deploy` 覆盖 `data/`。

```bash
sudo systemctl restart guideflow   # 改代码或 .env 后重启
sudo journalctl -u guideflow -f    # 看实时日志
```

## 15. 方案 B（可选）：nginx 反代，用 80 端口访问

目的：链接变成 `http://YOUR_EIP/`，不必写 `:8001`。

### 15.1 安装 nginx

```bash
sudo apt install -y nginx
```

### 15.2 配置反代

```bash
sudo nano /etc/nginx/sites-available/guideflow
```

示例（uvicorn 仍监听本机 8001）：

```nginx
server {
    listen 80;
    server_name YOUR_EIP;   # 有域名则写成域名

    client_max_body_size 20m;

    location / {
        proxy_pass http://127.0.0.1:8001/app/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8001/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;          # SSE 流式输出重要
        proxy_cache off;
        proxy_read_timeout 3600s;
    }

    location /health {
        proxy_pass http://127.0.0.1:8001/health;
    }

    location /app/ {
        proxy_pass http://127.0.0.1:8001/app/;
        proxy_set_header Host $host;
    }
}
```

> 上面把站点根路径指到 `/app/`；前端在 `/app/` 下会自动同源请求 `/api/...`。请保证浏览器访问的源与 Cookie 一致，并更新 `.env`：

```bash
CORS_ORIGINS=http://YOUR_EIP
```

启用配置：

```bash
sudo ln -sf /etc/nginx/sites-available/guideflow /etc/nginx/sites-enabled/guideflow
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
```

安全组放行 **80**。医生访问：`http://YOUR_EIP/`

有域名时：在域名解析里加 A 记录指向 `YOUR_EIP`，再把 `server_name`、`.env` 的 `CORS_ORIGINS`、发给医生的链接改成域名（前端走 `/app/` 时一般不用再改 `API_BASE`）。HTTPS 可用 Let’s Encrypt（`certbot`），上 HTTPS 后设 `COOKIE_SECURE=1`。

## 18. 常见问题

### `git clone` 报 GnuTLS recv error (-110) 或超时

- **原因**：国内云服务器访问 GitHub HTTPS 不稳定，与账号无关。  
- **推荐**：改用 **方式 B 本机打包上传**（见第 7 节），从你 Windows 电脑 `scp` 上去最稳。  
- **备选**：`git clone https://mirror.ghproxy.com/https://github.com/CHEN814/Guideflow.git .`（镜像站偶发不可用，可多试几次）。  
- 确认克隆命令末尾有 **`.`**，否则文件会在 `/opt/guideflow/Guideflow/` 多一层目录。

### 浏览器一直转圈 / 无法访问

- 安全组是否放行 8001（或 80）  
- `uvicorn` 是否使用 `--host 0.0.0.0`  
- `systemctl status guideflow` 是否 running  
- 本机能否 `curl http://YOUR_EIP:8001/health`

### 页面能开，但登录失败或提问报网络错误

- 是否通过 **`/app/`** 访问（推荐：`http://YOUR_EIP:8001/app/`）；不要只用错地址或旧缓存  
- `.env` 的 `CORS_ORIGINS` 是否包含浏览器地址栏里的源（含协议与端口）  
- `systemctl status guideflow` 是否 running；改完 `.env` 后是否 `systemctl restart guideflow`  
- 浏览器强刷（Ctrl+F5）排除旧 JS 缓存

### 回答提示知识库未就绪 / degraded

- 检查 `data/indexes/bm25_index.pkl` 与 `data/processed/` 是否存在  
- 看日志：`journalctl -u guideflow -n 100`

### SSH 能上，但 pip 很慢或失败

- 使用清华源（上文已写）  
- 磁盘 `df -h` 是否已满  

### 忘记登录密码

- 在华为云 ECS 控制台使用 **重置密码**，按指引操作后重启实例

### 公网 IP 变了怎么办？

- 按需计费释放重建后 IP 可能变化；需同步改：`.env` 的 `CORS_ORIGINS`、发给医生的链接（前端走 `/app/` 时一般不用改页面里的 API 地址）  
- 需要固定 IP 时可关注「弹性公网 IP」是否为独立持有/绑定（以控制台产品说明为准）
