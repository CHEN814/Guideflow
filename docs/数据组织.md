# 数据组织说明

> 离线构建步骤见 [`环境配置.md`](./环境配置.md)；在线检索与智能体流程见 [`技术实现.md`](./技术实现.md)。

## 来源 PDF 结构

```
NCCN B-Cell Lymphomas V3.2026
├── 页 1-13    前置部分（front_matter）
│              封面、疾病目录、更新日志
├── 页 14-139  临床实践指南（clinical_guideline）
│              每页一个决策流程单元，右下角有疾病页码标识
│              例如：BCEL-1、BCEL-A 1 OF 3、MANT-2
└── 页 140+    Discussion 部分
               多个疾病文章，每篇文章含正文和对应参考文献
               每页左上角有 MS-* 页码（如 MS-116）
```

## 知识库结构（JSON 文件）

输出文件：`data/processed/dlbcl_knowledge_base.json`

顶层字段：

```json
{
  "guideline_pages":    [...],
  "discussion_chunks":  [...],
  "reference_entries":  [...],
  "stats":              {...}
}
```

## 三类主对象

### 1. `GuidelinePage`

对应 PDF 每一页（前置页 + 临床指南页 + discussion 页）。

| 字段 | 说明 |
|------|------|
| `page_id` | 唯一 ID，例如 `page-14` |
| `pdf_page` | PDF 物理页码（1-based） |
| `page_type` | `front_matter` / `clinical_guideline` / `discussion_toc` / `discussion_text` / `discussion_references` |
| `clean_text` | 正则清洗后的页面全文（去除页眉、版权、版本号等噪声） |
| `printed_page_code` | 指南印刷页码，例如 `BCEL-1`、`MANT-A 1 OF 5`，仅临床指南页有 |
| `module_code` | 疾病/模块代码，从 `printed_page_code` 派生，例如 `BCEL`、`MANT` |
| `outgoing_links` | 页面跳转链接列表（见 PageLink），仅临床指南页有 |
| `needs_review` | 是否需要人工校验（footer code 未能抽取时为 `true`） |

**只有 `clinical_guideline` 类型的页面进入 BM25 检索索引**。`front_matter` 和 `discussion_*` 页面保留用于溯源；Discussion **正文 chunk** 单独进入索引（见 `DiscussionChunk`）。

### 2. `DiscussionChunk`

Discussion 正文按标题层级和语义段落切分。疾病 article 的边界由 NCCN 物理结构（正文 → 参考文献 → 正文）划分，病名仅用于贴标签；因此同一 article 的正文 chunk 与其后参考文献共享相同的 `article_id`，不会出现某疾病文献混入其它疾病的情况。目录页（`discussion_toc`）只用于解析章节标题清单，不产出疾病 chunk。

| 字段 | 说明 |
|------|------|
| `chunk_id` | 唯一 ID，例如 `disc-dlbcl-p255-c0` |
| `article_id` | 所属疾病 article，例如 `dlbcl`、`fl`、`mcl` |
| `article_title` | 疾病文章标题，例如 `Diffuse Large B-Cell Lymphoma` |
| `pdf_page` | 来源 PDF 页码 |
| `ms_page_code` | Discussion 内部页码，例如 `MS-116` |
| `section` | 小节标题，例如 `Treatment`、`Prognosis` |
| `clean_text` | 清洗后的段落文本，直接进检索索引 |
| `reference_ids` | 正文内引用编号列表，例如 `["45", "46", "47"]` |

### 3. `ReferenceEntry`

Discussion 每篇疾病 article 末尾的参考文献，按编号逐条解析。

| 字段 | 说明 |
|------|------|
| `entry_id` | 唯一 ID，例如 `ref-dlbcl-1` |
| `article_id` | 所属疾病 article（与 `DiscussionChunk` 中的 `article_id` 一致） |
| `ref_number` | 参考文献编号，例如 `"1"` |
| `text` | 参考文献全文 |
| `pmid` | PubMed ID（从正文链接或文本解析） |
| `doi` | DOI 编号 |
| `url` | 原始 URL（兜底字段） |

## 辅助结构

### `PageLink`

每条临床指南页的跳转链接，挂在 `GuidelinePage.outgoing_links` 下。

| 字段 | 说明 |
|------|------|
| `source_page_code` | 来源页面的 `printed_page_code` |
| `target_pdf_page` | 目标 PDF 物理页码 |
| `target_page_code` | 目标页面的 `printed_page_code`（反查得到） |
| `anchor_text` | 链接覆盖的文字（空间匹配得到） |

### `SearchDocument`

BM25 索引的最小单元，由 `StructuredKnowledgeBase.to_search_documents()` 生成。

| 字段 | 说明 |
|------|------|
| `source_id` | 唯一 ID，对应上游对象的 ID |
| `page_type` | `clinical_guideline` / `discussion` / `reference` |
| `pdf_page` | PDF 物理页码 |
| `text` | 检索文本 |
| `printed_page_code` | 临床指南页码（仅 `clinical_guideline` 有） |
| `module_code` | 疾病模块（仅 `clinical_guideline` 有） |
| `section` | 小节标题 |
| `article_id` | 疾病 article（`discussion`/`reference` 有） |
| `reference_ids` | 引用编号 |
| `needs_review` | 是否需要人工校验 |

### `EvidenceBundle`

问答阶段的证据包，由检索结果和参考文献关联展开组装。

| 字段 | 说明 |
|------|------|
| `primary_hits` | BM25 检索命中的指南页/讨论段落，用于 `[S1]`…`[Sn]` 编号 |
| `attached_references` | 由 discussion 的 `reference_ids` 关联展开的 `ReferenceEntry` 列表 |
| `reference_links` | `source_id -> [ref_number, ...]`，记录每条文献由哪些 discussion 块引用 |
| `figures` | 按需渲染的流程图引用（智能体 `view_pages` 或 linear 选图） |
| `graph_triples` | 知识图谱三元组（智能体 `query_graph` 或 linear 固定步骤） |

## 对象关系

```
GuidelinePage (page_type=clinical_guideline)
  └── outgoing_links: [PageLink, ...]
        └── target_page_code -> 另一个 GuidelinePage.printed_page_code

DiscussionChunk
  └── article_id ──┐
                   ├── 同一 article_id 的所有 chunks 和 references
ReferenceEntry        属于同一疾病文章
  └── article_id ──┘
  └── ref_number -> DiscussionChunk.reference_ids 中的编号

SearchDocument  ← 由上述对象生成
  └── page_type 决定检索路由

EvidenceBundle  ← 问答时组装
  ├── primary_hits: 来自 SearchDocument 检索结果（guideline + discussion）
  ├── figures: 来自按需渲页（view_pages / _gather_figures）
  └── attached_references: 由 ReferenceResolver 从知识库直查，不经 BM25 检索
```

## 检索索引来源

```
BM25 索引（data/indexes/bm25_index.pkl）
  ← SearchDocument 列表（由 kb.to_search_documents() 生成）
       ├── clinical_guideline 页面（clean_text）
       └── discussion chunks（语义段落）
```

说明：

- **`reference` 条目不进入 BM25 检索**，避免与 discussion 竞争 top-k。运行时通过 `ReferenceResolver` 从知识库 JSON 按 `(article_id, ref_number)` 直查展开。
- 运行时 `PageSummaryCache` 会把缓存的页摘要**内存并入** BM25 语料（不写回 pkl），提升流程图页召回。
- **无向量索引**；项目已移除 embedding / vector store 相关代码与构建脚本。

检索时 `MetadataFilters` 按 `page_type`、`module_code`、`article_id` 过滤：

| 路由标签 `route` | 触发关键词 | BM25 检索范围 |
|----------------|-----------|---------------|
| `flowchart` | 治疗路径类 | `clinical_guideline` + `discussion` |
| `evidence` | 预后/突变/证据类 | `clinical_guideline` + `discussion` |
| `hybrid` | 两类都有 | `clinical_guideline` + `discussion` |

> `route=hybrid` 仅表示问题同时含路径与证据关键词，**不是**向量 hybrid 检索。

疾病范围（默认 `disease_scope: auto`）叠加规则：

- `discussion`：按 `article_id` 过滤（如 `dlbcl`）
- `clinical_guideline`：按 `module_code` 过滤（如 `BCEL`）

### 与智能体的关系

默认 `routing_mode: agentic` 时，BM25 检索由工具 `search_guidelines` 触发，产出：

- `AgentState.hits` → 组装 `EvidenceBundle.primary_hits`
- `AgentState.candidate_pages` → 供 `view_pages` 点名的页码清单（含 seed 决策页、邻居页）

详见 [`技术实现.md`](./技术实现.md) 中「智能体工具循环」一节。

## 引用编号体系

问答中存在两套编号，职责不同：

| 编号 | 来源 | 用途 |
|------|------|------|
| `[S1]`…`[Sn]` | 检索 rerank 后的 primary hits 顺序编号 | 模型回答中的医学判断引用；CLI「证据来源」展示 |
| `[27]`、`[33]` 等 | `ReferenceEntry.ref_number`（NCCN 文献编号） | CLI「关联参考文献」展示；供读者查阅原文献 |
| `[G1]`… | 知识图谱三元组 | 图谱推理引用（prompt 与 trace） |

规则：

- 模型回答 **只能** 使用 `[Sn]`（及 prompt 允许的 `[Gn]`）引用指南页/讨论段落/图谱。
- 关联参考文献单独列出，不占用 `[Sn]` 编号，也不替代 `[Sn]` 做医学判断引用。
- discussion 块被检索命中后，其 `reference_ids` 经句子级筛选展开到输出（默认最多 6 条，见 `retrieval.max_attached_refs`）。

## 构建流程

```bash
# 1. 解析 PDF 生成结构化知识库
python scripts/build_knowledge_base.py

# 2. 构建 BM25 索引（唯一检索索引）
python scripts/build_bm25_index.py

# 3. （可选）构建知识图谱
python scripts/build_knowledge_graph.py
```

产出路径：

| 产物 | 路径 |
|------|------|
| 知识库 JSON | `data/processed/dlbcl_knowledge_base.json` |
| BM25 索引 | `data/indexes/bm25_index.pkl` |
| 知识图谱 | `data/processed/knowledge_graph.json` |

知识库或 BM25 语料变更后，需重新执行对应构建步骤。完整环境说明见 [`环境配置.md`](./环境配置.md)。

## 溯源路径

### 证据来源（primary hits）

检索命中 → `SearchDocument.source_id` → 上游对象 → PDF 位置

- `page-BCEL-1` → `printed_page_code == "BCEL-1"` 的 `GuidelinePage`
- `disc-dlbcl-p255-c0` → `DiscussionChunk`，含 `pdf_page`、`ms_page_code`、`section`

### 关联参考文献（attached references）

discussion 命中 → 句子级筛选 → `reference_ids` → `ReferenceResolver` 查表 → `ReferenceEntry`

- `disc-dlbcl-p140-c0` 含 `reference_ids=["27"]`
- 查得 `ref-dlbcl-27` → `ReferenceEntry(ref_number="27", text="Dodero A, et al. ...")`
- CLI 展示为 `[27] Dodero A, et al. ... (由 S1 关联)`

关联展开不依赖索引检索，直接读取知识库 JSON；默认最多 `max_attached_refs`（6）条，超出在 trace 中标记截断。

### 流程图（figures）

智能体 `view_pages` 或 linear `_gather_figures` → `printed_page_code` → `GuidelinePage.pdf_page` → 按需渲染 PNG → `data/cache/page_images/`
