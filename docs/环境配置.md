# 环境配置指南

本文档说明从零开始配置本项目的完整开发/运行环境，包括 Python 虚拟环境、依赖安装、Hugging Face 镜像、本地模型下载、`.env`（密钥）与 `config.yaml`（非密钥）配置、索引构建与验证。

适用场景：

- 首次在本机部署项目
- 从「降级模式」（hash embedding / 词面 reranker）切换到真实向量模型
- 国内网络访问 Hugging Face 不稳定，需要使用镜像

---

## 1. 环境要求

### 1.1 软件

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、Linux 或 macOS（下文以 **Windows PowerShell** 为主，其他系统见对应注释） |
| Python | **3.10 ~ 3.12**（推荐 3.11）。运行 `python --version` 确认 |
| Git | 可选，用于克隆仓库 |
| 网络 | 首次下载模型需联网；可使用 Hugging Face 镜像 |

### 1.2 硬件

| 项目 | 最低建议 | 说明 |
|------|----------|------|
| 内存 | 8 GB（推荐 16 GB） | bge-m3 加载后约占 4~6 GB RAM |
| 磁盘 | 至少 5 GB 空闲 | 模型约 2.3 GB + reranker 约 1 GB + 索引与知识库 |
| 显卡 | **非必须** | 纯 CPU 可完成全部任务；有 NVIDIA GPU 时会自动加速 |

### 1.3 你需要准备的内容

1. 项目源码目录（含 `requirements.txt`、`scripts/` 等）
2. NCCN B 细胞淋巴瘤 PDF（放在项目根目录，或通过 `.env` 指定路径）
3. 千问（Qwen）API Key（仅正式问答 `ask.py` 需要；检索调试不需要）

## 3. 创建 Python 虚拟环境

**强烈建议使用项目独立 venv**，避免与 Anaconda / 系统 Python 的 numpy、torch 版本冲突。

```powershell
python -m venv .venv
```

激活虚拟环境：
```bash
cd "D:\Project\guideflow\code"
.\.venv\Scripts\Activate.ps1
```


```powershell
# Windows PowerShell

```

若 PowerShell 提示「无法加载，因为在此系统上禁止运行脚本」，先执行（仅当前用户、一次性）：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新激活。

```bash
# Linux / macOS
source .venv/bin/activate
```

激活成功后，命令行前缀会出现 `(.venv)`。

验证当前 Python 来自 venv：

```powershell
python -c "import sys; print(sys.executable)"
```

应输出类似：

```text
...\code\.venv\Scripts\python.exe
```

## 4. 配置国内镜像（推荐）

国内访问 `huggingface.co` 经常超时或失败。建议在**安装依赖和下载模型之前**配置镜像。

### 4.1 Hugging Face 模型镜像（必配）

本项目 embedding / reranker 模型通过 `sentence-transformers` 从 Hugging Face 下载。设置环境变量 `HF_ENDPOINT` 即可走镜像站。

**PowerShell（当前终端会话有效）：**

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
```

**CMD：**

```cmd
set HF_ENDPOINT=https://hf-mirror.com
```

**Linux / macOS：**

```bash
export HF_ENDPOINT=https://hf-mirror.com
```

**持久化（Windows 用户环境变量，推荐）：**

1.  Win + R → 输入 `sysdm.cpl` →「高级」→「环境变量」
2.  在「用户变量」中新建：
    - 变量名：`HF_ENDPOINT`
    - 变量值：`https://hf-mirror.com`
3.  重启终端后生效

验证：

```powershell
echo $env:HF_ENDPOINT
```

### 4.2 pip 镜像（可选，加速依赖安装）

```powershell
python -m pip install -U pip
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple
```

也可单次安装时指定：

```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 4.3 镜像使用注意

- `HF_ENDPOINT` 必须在**运行会下载模型的命令之前**设置好（如 `build_vector_index.py`、`inspect_retrieval.py`）
- 镜像只替代 Hugging Face 下载地址，不影响 Qwen / VLM API 调用
- 若镜像也失败，见本文第 8 节「手动下载模型到本地」

## 5. 安装依赖

`requirements.txt` 包含项目全部依赖：PDF 解析、BM25、Qwen 调用，以及 embedding / reranker（torch、sentence-transformers、chromadb 等）。

在已激活的 venv 中执行：

```powershell
pip install -r requirements.txt
```

首次安装 torch 体积较大（约 1~2 GB），请耐心等待。

### 5.1 验证依赖

```powershell
python -c "import torch; print('torch', torch.__version__, 'CUDA:', torch.cuda.is_available())"
python -c "import sentence_transformers; print('sentence-transformers OK')"
python -c "import chromadb; print('chromadb OK')"
```

说明：

- `CUDA: False` 表示使用 CPU，**完全正常**，不影响功能
- `CUDA: True` 表示检测到 NVIDIA GPU，模型推理会自动加速

### 5.2 常见安装问题

**numpy 版本冲突（多与 Anaconda 混用有关）**

- 确保已激活 `.venv`，不要用 Anaconda 的 `base` 环境
- 若仍报错，删除 `.venv` 文件夹后从第 3 节重来

**torch 安装失败**

- 确认 Python 版本在 3.10~3.12
- 可访问 [PyTorch 官网](https://pytorch.org/get-started/locally/) 按系统选择安装命令，再安装其余依赖

## 6. 配置 `.env` 与 `config.yaml`

### 6.1 分工（业内常规做法）

| 文件 | 放什么 | 是否提交 Git |
|------|--------|--------------|
| `.env` | API Key 等**密钥** | 否 |
| `config.yaml` | 路径、模型名、TopK、疾病范围、裁剪开关等**非密钥** | 是 |

加载顺序：**环境变量 > config.yaml > 代码默认值**。临时覆盖仍可用环境变量（如 `FINAL_TOP_K=8`）。

### 6.2 `.env`（仅密钥）

```text
QWEN_API_KEY=你的_DashScope_千问_API_Key
VLM_API_KEY=你的_DashScope_Key
# 可选：
# QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
# QWEN_MODEL=qwen-plus
```

### 6.3 `config.yaml`（非密钥，见项目根目录完整示例）

```yaml
paths:
  pdf: "（2026.V3）NCCN临床 实践指南：B细胞淋巴瘤.pdf"
  knowledge_base: data/processed/dlbcl_knowledge_base.json
disease_scope: dlbcl
retrieval:
  mode: bm25
  vector_top_k: 12
  rerank_top_k: 16
  final_top_k: 6
embedding:
  model: hash
reranker:
  model: lexical
crop:
  enabled: true
  prefer: auto
  padding: 0.02
  min_area: 0.05
```

### 6.4 关键变量说明

| 变量 / 配置项 | 默认值 | 说明 |
|------|--------|------|
| `paths.pdf` / `NCCN_PDF_PATH` | 根目录第一个 PDF | PDF 路径（不要用已废弃的 `GUIDELINE_PDF_PATH`） |
| `embedding.model` / `EMBEDDING_MODEL` | `hash` | 设为 `BAAI/bge-m3` 启用真实向量 |
| `reranker.model` / `RERANKER_MODEL` | `lexical` | 设为 `BAAI/bge-reranker-v2-m3` 启用 CrossEncoder |
| `disease_scope` / `TARGET_DISEASE_SCOPE` | `dlbcl` | 疾病范围，见 `backend/app/services/disease_scope.py` |
| `crop.enabled` | `true` | 是否为展示生成裁剪框图（默认 `crop.prefer: auto`，PyMuPDF 确定性几何优先，VLM bbox 仅兜底） |
| `crop.prefer` | `auto` | `auto` \| `vlm` \| `detect`；`auto`=表格/矢量/文本检测链，`vlm`=调试 VLM 兜底，`detect`=仅矢量图 |

提示词模板集中在 `backend/app/prompts.py`。

### 6.5 仅 CPU、想先跳过重排

在 `config.yaml` 中保持：

```yaml
reranker:
  model: lexical
```

embedding 仍使用 bge-m3，检索主体质量不受影响，仅重排步骤降级为词面匹配。

## 7. 下载模型

模型不会在 `pip install` 时下载，而是在**首次加载**时从 Hugging Face（或镜像）拉取。

| 模型 | Hugging Face ID | 约占用磁盘 |
|------|-----------------|------------|
| Embedding | `BAAI/bge-m3` | ~2.3 GB |
| Reranker | `BAAI/bge-reranker-v2-m3` | ~1.1 GB |

### 7.1 方式 A：随建索引自动下载（推荐）

确保已设置镜像并激活 venv：

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
python scripts/build_vector_index.py
```

首次运行会看到下载进度，完成后模型缓存在：

```text
Windows: C:\Users\<用户名>\.cache\huggingface\hub\
Linux:   ~/.cache/huggingface/hub/
```

### 7.2 方式 B：提前单独验证下载

不建索引，仅测试模型能否加载：

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
python -c "from sentence_transformers import SentenceTransformer; m=SentenceTransformer('BAAI/bge-m3'); print('dim:', m.encode(['test']).shape)"
```

期望输出：

```text
dim: (1, 1024)
```

bge-m3 输出向量维度为 **1024**。

Reranker 验证（可选）：

```powershell
python -c "from sentence_transformers import CrossEncoder; m=CrossEncoder('BAAI/bge-reranker-v2-m3'); print('score:', m.predict([('问题','文档')]))"
```

### 7.3 方式 C：手动下载到项目目录（离线 / 镜像仍失败时）

```powershell
pip install huggingface_hub

$env:HF_ENDPOINT = "https://hf-mirror.com"

huggingface-cli download BAAI/bge-m3 --local-dir models/bge-m3
huggingface-cli download BAAI/bge-reranker-v2-m3 --local-dir models/bge-reranker-v2-m3
```

然后在 `.env` 中改为本地路径：

```text
EMBEDDING_MODEL=./models/bge-m3
RERANKER_MODEL=./models/bge-reranker-v2-m3
```

建议将 `models/` 加入 `.gitignore`（若尚未忽略），避免误提交大文件。

## 8. 构建数据与索引（逐步执行）

按顺序执行以下命令。每一步成功后再进行下一步。

### 步骤 1：解析 PDF，生成知识库

```powershell
python scripts/build_knowledge_base.py
```

产出：`data/processed/dlbcl_knowledge_base.json`

若报找不到 PDF，检查 `.env` 中的路径或把 PDF 放到项目根目录。

### 步骤 2：构建 BM25 索引

```powershell
python scripts/build_bm25_index.py
```

产出：`data/indexes/bm25_index.pkl`

### 步骤 3：构建向量索引（触发 bge-m3 下载与 embedding）

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
python scripts/build_vector_index.py
```

成功时终端输出类似：

```text
Vector index written to: data\indexes\vector
Indexed documents: 1234
```

**重要：**

- 输出中**不应**出现 `Degraded: hash_embedding_fallback`
- 若出现，说明 embedding 模型未正确加载，见第 10 节排查
- 若之前用 hash 模式建过索引，**必须**重新执行本步骤

CPU 上建索引可能需要数分钟到十几分钟，属正常现象。

### 步骤 4：验证检索（不消耗 Qwen API）

```powershell
$env:HF_ENDPOINT = "https://hf-mirror.com"
python scripts/inspect_retrieval.py "TP53 R248Q 在 DLBCL 的预后意义是什么？" --trace
```

检查：

1. 终端列出若干条 `Final evidence`，分数与来源合理
2. 末尾**没有** `Degraded:` 段，或 `Degraded` 中不含 `hash_embedding_fallback`、`reranker_disabled_lexical_fallback`
3. 若加了 `--trace`，打开 `logs/runs/` 下最新 jsonl 文件查看中间过程

### 步骤 5：正式问答（需要 Qwen API Key）

```powershell
python scripts/ask.py "TP53 R248Q 在 DLBCL 的预后意义是什么？" --trace
```

### 步骤 6：运行测试（可选）

```powershell
python -m pytest -q
```

## 10. 故障排查

### 10.1 出现 `hash_embedding_fallback`

**含义：** 未使用 bge-m3，已降级为 hash 向量，检索质量明显下降。

**排查：**

1. `.env` 中是否设置了 `EMBEDDING_MODEL=BAAI/bge-m3`
2. venv 中是否安装了 ML 依赖：`pip show sentence-transformers torch`
3. 模型是否下载成功：检查 `%USERPROFILE%\.cache\huggingface\hub` 或运行第 7.2 节验证命令
4. 下载失败时确认 `HF_ENDPOINT` 已设置
5. 修复后**重建向量索引**

### 10.2 出现 `reranker_disabled_lexical_fallback`

**含义：** CrossEncoder 未加载，重排降级为词面匹配。

**排查：** 同 embedding，检查 `RERANKER_MODEL` 与 `sentence-transformers`；或暂时设 `RERANKER_MODEL=lexical` 接受降级。

### 10.3 Hugging Face 下载超时 / SSL / 连接重置

1. 确认 `$env:HF_ENDPOINT = "https://hf-mirror.com"`
2. 使用第 7.3 节手动下载到 `models/`
3. 关闭 VPN 或切换网络后重试（部分 VPN 与镜像冲突）

### 10.4 检索结果明显不对

常见原因：**索引与 query 使用了不同的 embedding 空间**（例如索引用 hash 建的，query 用 bge-m3）。

处理：删除 `data/indexes/vector/` 目录后重新 `python scripts/build_vector_index.py`。

### 10.5 内存不足 / 进程被系统杀死

- 关闭其他占内存程序
- 暂时设 `RERANKER_MODEL=lexical` 减少同时加载的模型
- 确保物理内存 ≥ 8 GB

### 10.6 Qwen API 报错

- 检查 `QWEN_API_KEY` 是否正确、是否有余额
- `inspect_retrieval.py` 不调用 API，可单独用来确认检索链路正常

## 11. 配置层级与降级行为

本项目采用「优先真实模型、失败则降级」策略：

```
EMBEDDING_MODEL=BAAI/bge-m3
  ├─ sentence-transformers + torch 可用 → 真实 1024 维向量
  └─ 加载失败 → HashEmbedding（trace: hash_embedding_fallback）

RERANKER_MODEL=BAAI/bge-reranker-v2-m3
  ├─ CrossEncoder 可用 → 神经重排
  └─ 加载失败 → LexicalReranker（trace: reranker_disabled_lexical_fallback）

chromadb 可用 → ChromaDB 持久化向量库
  └─ 不可用 → 本地 JSON 向量索引（功能等价，质量取决于 embedding）
```

因此：**环境配置是否成功的金标准，是 trace / 终端输出中没有上述 fallback 标记。**
