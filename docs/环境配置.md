# 环境配置指南

本文档说明从零开始配置本项目的开发/运行环境：Python 虚拟环境、依赖安装、`.env`（密钥）与 `config.yaml`（非密钥）、**BM25 索引**构建与验证。

适用场景：

- 首次在本机部署项目
- 更新知识库或 PDF 后重建 BM25 索引
- 调整智能体路由、看图预算等在线参数

架构细节见 [`技术实现.md`](./技术实现.md)；数据字段见 [`数据组织.md`](./数据组织.md)。

---

## 1. 环境要求

### 1.1 软件

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、Linux 或 macOS（下文以 **Windows PowerShell** 为主） |
| Python | **3.10 ~ 3.12**（推荐 3.11）。运行 `python --version` 确认 |
| Git | 可选，用于克隆仓库 |
| 网络 | 正式问答需访问 DashScope（Qwen / VLM API）；离线构建仅需本地 PDF |

### 1.2 硬件

| 项目 | 最低建议 | 说明 |
|------|----------|------|
| 内存 | 8 GB（推荐 16 GB） | BM25 索引与 PDF 解析占用较低；无本地 embedding 模型 |
| 磁盘 | 至少 2 GB 空闲 | 知识库 JSON + BM25 pkl + 页图缓存 |
| 显卡 | **非必须** | 本项目不依赖 GPU / torch |

### 1.3 你需要准备的内容

1. 项目源码目录（含 `requirements.txt`、`scripts/` 等）
2. NCCN B 细胞淋巴瘤 PDF（放在项目根目录，或在 `config.yaml` 的 `paths.pdf` 指定）
3. 千问（Qwen）API Key（正式问答需要；`inspect_retrieval.py` 仅检索调试不需要）
4. VLM API Key（可选；流程图「看图回答」需要，可与 Qwen 共用 DashScope Key）

## 3. 创建 Python 虚拟环境

**强烈建议使用项目独立 venv**，避免与系统 Python 版本冲突。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

若 PowerShell 禁止运行脚本：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

Linux / macOS：

```bash
source .venv/bin/activate
```

验证：

```powershell
python -c "import sys; print(sys.executable)"
```

应指向 `.venv` 下的 `python.exe`。

**不包含** torch、sentence-transformers、chromadb（向量检索已移除）。

```powershell

```

## 6. 配置 `.env` 与 `config.yaml`

### 6.1 分工

| 文件 | 放什么 | 是否提交 Git |
|------|--------|--------------|
| `.env` | API Key 等**密钥** | 否 |
| `config.yaml` | 路径、TopK、智能体参数、裁剪开关等**非密钥** | 是 |

加载顺序：**环境变量 > config.yaml > 代码默认值**。

### 6.2 `.env`（仅密钥）

```text
QWEN_API_KEY=你的_DashScope_千问_API_Key
VLM_API_KEY=你的_DashScope_Key
# 可选覆盖：
# QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
# QWEN_MODEL=qwen-plus
# VLM_MODEL=qwen-vl-max
```

以 **UTF-8 with BOM** 或 UTF-8 保存均可（代码用 `utf-8-sig` 读取）。

### 6.3 `config.yaml` 示例（与仓库一致）

```yaml
paths:
  pdf: "（2026.V3）NCCN临床 实践指南：B细胞淋巴瘤.pdf"
  knowledge_base: data/processed/dlbcl_knowledge_base.json
  bm25_index: data/indexes/bm25_index.pkl
  knowledge_graph: data/processed/knowledge_graph.json
  page_images: data/cache/page_images
  summary_cache: data/cache/page_summaries.json
  logs: logs/runs

qwen:
  base_url: https://dashscope.aliyuncs.com/compatible-mode/v1
  model: qwen-plus

vlm:
  base_url: https://dashscope.aliyuncs.com/compatible-mode/v1
  model: qwen-vl-max

reranker:
  model: lexical

retrieval:
  bm25_top_k: 40
  rerank_top_k: 16
  final_top_k: 6
  max_attached_refs: 6

disease_scope: auto

routing_mode: agentic
agent_max_steps: 4

figure_ceiling: 4
max_images: 4
display_max_figures: 4
enable_evidence_gating: true

graph:
  fanout: 3
  depth: 1
  reserve: 2

crop:
  enabled: true
  prefer: auto
  padding: 0.02
  min_area: 0.05
```

### 6.4 关键配置说明

| 配置项 / 环境变量 | 默认 | 说明 |
|-------------------|------|------|
| `paths.pdf` / `NCCN_PDF_PATH` | 根目录第一个 PDF | 指南 PDF 路径 |
| `paths.bm25_index` | `data/indexes/bm25_index.pkl` | 唯一检索索引 |
| `reranker.model` / `RERANKER_MODEL` | `lexical` | 词面重排（无需额外模型） |
| `retrieval.*` | 见上 | BM25 / rerank / 最终 top-k、参考文献上限 |
| `disease_scope` / `TARGET_DISEASE_SCOPE` | `auto` | 病种过滤，见 `disease_scope.py` |
| `routing_mode` / `ROUTING_MODE` | `agentic` | `agentic`=工具循环；`linear`=旧线性流水线 |
| `agent_max_steps` / `AGENT_MAX_STEPS` | `4` | 智能体最大工具轮数 |
| `figure_ceiling` / `FIGURE_CEILING` | `4` | 按需看图上限（与 `max_images` 联动） |
| `enable_evidence_gating` | `true` | 证据门控开关 |
| `crop.*` | 见上 | 展示裁剪；`prefer: auto` 为确定性几何优先 |

提示词模板集中在 `backend/app/prompts.py`。

## 7. 构建数据与索引

按顺序执行。每一步成功后再进行下一步。

### 步骤 1：解析 PDF，生成知识库

```powershell
python scripts/build_knowledge_base.py
```

产出：`data/processed/dlbcl_knowledge_base.json`

若报找不到 PDF，检查 `config.yaml` 的 `paths.pdf` 或把 PDF 放到项目根目录。

### 步骤 2：构建 BM25 索引

```powershell
python scripts/build_bm25_index.py
```

产出：`data/indexes/bm25_index.pkl`

**无向量索引步骤。** 若曾存在 `data/indexes/vector/` 等旧目录，可安全删除。

### 步骤 3：（可选）构建知识图谱

```powershell
python scripts/build_knowledge_graph.py
```

产出：`data/processed/knowledge_graph.json`。缺失时在线问答自动降级为空图。

### 步骤 4：验证检索（不消耗 Qwen API）

```powershell
python scripts/inspect_retrieval.py "TP53 R248Q 在 DLBCL 的预后意义是什么？" --trace
```

检查终端列出的 `Final evidence` 是否合理；`--trace` 时查看 `logs/runs/` 下最新 jsonl。

### 步骤 5：正式问答（需要 Qwen API Key）

```powershell
python scripts/ask.py "TP53 R248Q 在 DLBCL 的预后意义是什么？" --trace
```

### 步骤 6：启动 Web 服务（可选）

```powershell
python -m uvicorn backend.api.server:app --reload --host 127.0.0.1 --port 8001
```

前端静态服务见 [`frontend/开发指南.md`](../frontend/开发指南.md)。

### 步骤 7：运行测试（可选）

```powershell
python -m pytest -q
```

常用命令：

```powershell
# 仅看 BM25 检索
python scripts/inspect_retrieval.py "你的问题" --trace

# 完整问答（默认 agentic）
python scripts/ask.py "你的问题" --trace

python scripts/ask.py "你的问题" --trace
```

修改知识库 JSON 后需重新执行 `build_bm25_index.py`。修改 PDF 后需从 `build_knowledge_base.py` 重来。

## 9. 故障排查

### 9.1 检索结果明显不对

- 确认 `data/indexes/bm25_index.pkl` 存在且与当前知识库 JSON 同步重建
- 检查 `disease_scope` 是否过窄（可临时设 `all` 对比）
- 用 `inspect_retrieval.py --trace` 查看 `retrieval_topk_raw` / `rerank_topk`

### 9.2 智能体不调用工具 / 延迟过高

- 确认 `QWEN_API_KEY` 有效；无 Key 时会 `fallback` 到单次检索
- 降低 `agent_max_steps` 或 `figure_ceiling` 控制延迟
- 设 `routing_mode: linear` 对比行为

### 9.3 流程图不显示 / 不读图

- 检查 `VLM_API_KEY`（或 `DASHSCOPE_API_KEY`）
- trace 中查看 `multimodal_decision`：`use_vlm` 是否为 true、`figure_count` 是否 > 0
- agentic 模式下问题是否触发了 `view_pages`（或回退 `_gather_figures`）

### 9.4 Qwen API 报错

- 检查 Key 是否正确、账户是否有余额
- `inspect_retrieval.py` 不调用 API，可单独确认 BM25 链路

### 9.5 页图 404

- 确认 `data/cache/page_images/` 可写；首次看图会按需生成 PNG
- Web 通过 `/api/images/{filename}` 提供，文件名需与 payload 中 `image_url` 一致

## 10. 完整一键流程（复制执行）

以下假设已在项目根目录，且 PDF 已就位。

```powershell
# 1. 创建并激活 venv
python -m venv .venv
.\.venv\Scripts\Activate.ps1

# 2. 安装依赖
pip install -U pip
pip install -r requirements.txt

# 3. 配置 .env（手动编辑，填入 API Key）

# 4. 构建流水线（无向量步骤）
python scripts/build_knowledge_base.py
python scripts/build_bm25_index.py
python scripts/build_knowledge_graph.py

# 5. 验证
python scripts/inspect_retrieval.py "TP53 R248Q 在 DLBCL 的预后意义是什么？" --trace
python scripts/ask.py "DLBCL 一线治疗方案是什么？" --trace

# 6. 启动 API
python -m uvicorn backend.api.server:app --host 127.0.0.1 --port 8001
```
