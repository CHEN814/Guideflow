# Guideflow 运维手册

面向**已完成首次上云部署**之后的日常运维：服务启停、日志、数据库查看、代码/数据更新、排障速查。

首次买机、装环境、配置 systemd → 见 [华为云部署指南](./华为云部署指南.md)。  
本机开发环境 → 见 [环境配置](./环境配置.md)。

下文默认：

- 项目目录：`/opt/guideflow`
- 服务名：`guideflow`（systemd）
- Web 应用库：`/opt/guideflow/data/app.db`（SQLite，可用 `DATABASE_URL` 覆盖）
- 公网访问：`http://YOUR_EIP:8001/app/`

---

## 1. 服务常用命令

SSH 登录服务器后：

```bash
# 状态
sudo systemctl status guideflow

# 启 / 停 / 重启（改代码、.env、data 后一般要重启）
sudo systemctl start guideflow
sudo systemctl stop guideflow
sudo systemctl restart guideflow

# 开机自启
sudo systemctl enable guideflow
sudo systemctl disable guideflow

# 最近日志 / 实时跟踪
sudo journalctl -u guideflow -n 200 --no-pager
sudo journalctl -u guideflow -f
sudo journalctl -u guideflow --since "1 hour ago" --no-pager
```

健康检查（本机或服务器上均可）：

```bash
curl -s http://127.0.0.1:8001/health
# 或：curl -s http://YOUR_EIP:8001/health
# 期望类似：{"status":"ok"}
```

若用了 nginx 方案 B：

```bash
sudo nginx -t
sudo systemctl status nginx
sudo systemctl reload nginx
sudo journalctl -u nginx -n 100 --no-pager
```

## 2. 查看应用数据库（SQLite）

用户、会话、对话消息存在 **`data/app.db`**（不是知识库 BM25 索引）。

### 2.1 安装客户端并打开

```bash
sudo apt install -y sqlite3   # 若尚未安装
cd /opt/guideflow
sqlite3 data/app.db
```

进入后提示符为 `sqlite>`。退出：`.quit` 或 `Ctrl+D`。

推荐先开表头与列模式：

```sql
.headers on
.mode column
.tables
.schema
```

### 2.2 主要表

| 表名 | 内容 |
|------|------|
| `users` | 注册用户（邮箱、密码哈希、创建时间） |
| `auth_sessions` | 登录会话（token 哈希、过期时间） |
| `conversations` | 对话列表（标题、归属用户、`active_root_id` 当前树根） |
| `messages` | 消息树节点（role、正文、payload_json、feedback、`parent_id`、`active_child_id`） |
| `shares` | 分享链接 token |

消息为**树形**存储：`parent_id` 指向上一节点（根为 NULL）；`active_child_id` / `conversations.active_root_id` 记录当前激活分支。编辑/重说会产生兄弟节点，而非覆盖。

**不要**在日志/聊天里贴出 `password_hash`、`token_hash` 等敏感字段。

### 2.3 常用查询示例

```sql
-- 用户列表（不含密码）
SELECT id, email, created_at FROM users ORDER BY created_at DESC;

-- 用户数
SELECT COUNT(*) AS user_count FROM users;

-- 某用户的对话
SELECT c.id, c.title, c.updated_at
FROM conversations c
JOIN users u ON u.id = c.user_id
WHERE u.email = 'someone@example.com'
ORDER BY c.updated_at DESC;

-- 最近对话
SELECT id, user_id, title, updated_at
FROM conversations
ORDER BY updated_at DESC
LIMIT 20;

-- 某对话的消息树（把 CONV_ID 换成真实 id）
SELECT id, parent_id, active_child_id, role,
       substr(content, 1, 80) AS preview, feedback, created_at
FROM messages
WHERE conversation_id = 'CONV_ID'
ORDER BY created_at;

-- 会话激活根
SELECT id, title, active_root_id, updated_at
FROM conversations
WHERE id = 'CONV_ID';

-- 点赞/点踩统计
SELECT feedback, COUNT(*) FROM messages
WHERE feedback IS NOT NULL
GROUP BY feedback;

-- 分享链接
SELECT token, conversation_id, created_at FROM shares ORDER BY created_at DESC LIMIT 20;
```

一行命令（不进入交互壳）：

```bash
sqlite3 -header -column /opt/guideflow/data/app.db \
  "SELECT id, email, created_at FROM users ORDER BY created_at DESC;"
```

### 2.4 备份与注意

```bash
# 热备一份（服务可继续跑；重要变更前建议先停服务再拷）
cp /opt/guideflow/data/app.db /opt/guideflow/data/app.db.bak.$(date +%Y%m%d)

# 或停服务后拷贝更稳妥
sudo systemctl stop guideflow
cp /opt/guideflow/data/app.db ~/app.db.bak
sudo systemctl start guideflow
```

- `ecs-deploy` / rsync **不会**覆盖远端 `data/`（含 `app.db`），本地同步代码不会冲掉线上用户数据。
- 不要把生产 `app.db` 提交到 Git 或发到群里。
- **含消息树字段的版本上线**：先按上面备份 `app.db`，再部署并 `systemctl restart guideflow`。启动时 `init_db()` 会幂等 `ALTER TABLE` 增加 `parent_id` / `active_child_id` / `active_root_id`，并把旧线性消息一次性回填为单链树（数据不丢）。异常时用 `.bak` 覆盖回滚后重启。


## 4. 配置与密钥

```bash
# 查看（勿把输出贴到公开场合）
grep -E '^(CORS_|AUTH_|COOKIE_|QWEN_|VLM_|DATABASE_)' /opt/guideflow/.env

# 改完 .env 必须重启
sudo nano /opt/guideflow/.env
sudo systemctl restart guideflow
```

公网 IP 变更后：同步改 `.env` 的 `CORS_ORIGINS` 与发给试用者的链接；前端走 `/app/` 时一般不用改页面里的 API 地址。

## 5. 更新代码（部署完成后）

### 5.1 推荐：本机 `ecs-deploy/` 一键同步

工具放在仓库内 `ecs-deploy/`，已 `.gitignore`，**不会**进 GitHub，同步时也**不会**传到云服务器。  
用 rsync：增量对照本地与远端；默认 `--delete` 会删掉云上本地已不存在的代码文件；**不会**覆盖远端 `.env`、`data/`、`.venv/`。

```powershell
cd D:\Project\guideflow\code\ecs-deploy
# 首次：Copy-Item .\config.example.ps1 .\config.ps1 并填入公网 IP

.\deploy.ps1 -Inventory   # 只读列出远端文件
.\deploy.ps1 -DryRun      # 模拟将上传/删除的文件
.\deploy.ps1              # 正式同步并 systemctl restart guideflow
```

前置：本机可 SSH；安装 rsync（WSL 内 `apt install rsync`，或 Windows PATH 中的 rsync）。细节见仓库内 `ecs-deploy/README.md`。

### 5.2 手动：传单个文件

```powershell
scp .\frontend\app.js root@YOUR_EIP:/opt/guideflow/frontend/app.js
scp .\backend\api\server.py root@YOUR_EIP:/opt/guideflow/backend/api/server.py
```

```bash
sudo systemctl restart guideflow   # 改了后端需要；只改静态前端有时强刷即可
```

### 5.3 手动：打一小包

```powershell
tar -czf patch.tgz frontend/app.js backend/api/some_file.py
scp .\patch.tgz root@YOUR_EIP:/opt/guideflow/
```

```bash
cd /opt/guideflow
tar -xzf patch.tgz
sudo systemctl restart guideflow
```

### 5.4 整包重传

按 [华为云部署指南 · 方式 B](./华为云部署指南.md) 打 `guideflow-src.tgz`（排除 `.venv`、`ecs-deploy`、`docs/临时文档` 等）。建议同时排除 `data/`，避免覆盖线上库与索引。若改了 `requirements.txt`：

```bash
cd /opt/guideflow
source .venv/bin/activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
sudo systemctl restart guideflow
```

注意：`tar`/`scp` 覆盖时**包里没有的文件不会被删**；不要把本地 `.env` 打进包。

## 6. 更新知识库 `data/`

问答依赖 `data/processed/`、`data/indexes/`（至少 `bm25_index.pkl`），与 `app.db` 分开。

本机已构建好时：

```powershell
scp -r .\data\processed root@YOUR_EIP:/opt/guideflow/data/
scp -r .\data\indexes root@YOUR_EIP:/opt/guideflow/data/
```

或在服务器上重建（见 [环境配置](./环境配置.md)）：

```bash
cd /opt/guideflow
source .venv/bin/activate
python scripts/build_knowledge_base.py
python scripts/build_bm25_index.py
sudo systemctl restart guideflow
```

上传/重建后务必 `restart`，并确认索引文件存在（见第 3 节）。

## 7. 费用与安全（日常）

| 操作 | 做法 |
|------|------|
| 临时省钱 | ECS 控制台 **关机**（按需计费是否仍收盘/IP 费以华为云说明为准） |
| 不用了 | 释放 ECS / 解绑并释放弹性公网 IP |
| API 费用 | Qwen / VLM 按调用计费，与是否买 GPU 无关 |
| SSH | 正式环境应收紧安全组 22 的来源 IP |
| 密钥 | 不要把 `.env`、`app.db`、密码发到群或截图外传 |

## 8. 运维排障速查

| 现象 | 检查 |
|------|------|
| 浏览器打不开 | 安全组端口；`systemctl status guideflow`；`curl .../health`；`ss -lntp \| grep 8001` |
| 页面能开，登录/提问失败 | 是否走 `/app/`；`CORS_ORIGINS` 是否含当前源；改 `.env` 后是否重启；浏览器强刷 |
| 知识库未就绪 / degraded | `bm25_index.pkl`、`data/processed/`；`journalctl -u guideflow -n 100` |
| pip / 磁盘问题 | `df -h`；清华源安装依赖 |
| 忘记 root 密码 | 华为云 ECS 控制台 **重置密码** 后重启实例 |
| 公网 IP 变了 | 改 `CORS_ORIGINS` 与对外链接 |

更完整的首次部署排障见 [华为云部署指南 · 常见问题](./华为云部署指南.md)。
